# Task 0004: Auth — register/login/logout

**Branch**: `feature/auth`
**Depends on**: 0001
**Source**: talk-it-through 2026-07-07 · **User stories**: lightweight account creation and login

## What to build

Account creation and session-based login for players — username + password only, no email — with
sessions persisted server-side in SQLite so logins survive server restarts, plus minimal React
screens to register, log in, and log out.

## AFK tasks

- [ ] Create SQLite schema/migration for a `users` table (id, username, password hash, created_at)
- [ ] Implement password hashing (e.g. bcrypt) — never store plaintext passwords
- [ ] Implement `POST /api/register` (username + password, enforcing unique username)
- [ ] Implement `POST /api/login` (validates credentials, establishes a server-side session)
- [ ] Implement `POST /api/logout` (destroys the session)
- [ ] Wire session storage to persist in SQLite (not just in-memory) so sessions survive server
      restarts
- [ ] Add session-based auth middleware to protect subsequent game endpoints
- [ ] Build minimal React register/login/logout UI wired to these endpoints, showing logged-in
      state
- [ ] Add automated integration tests for register/login/logout (valid + invalid credentials,
      duplicate username, session persistence)

## Acceptance criteria

- [ ] A user can register a username/password, log in, and the client reflects a logged-in state
- [ ] Passwords are stored only as hashes
- [ ] Logging out clears the session and protected endpoints reject unauthenticated requests
- [ ] Sessions survive a server restart (SQLite-backed store)
